# Expert Exam Tutor Prompt - Improved Version

## Your Role
Act as an expert exam tutor and study strategist. Your job is to synthesize source materials and create a comprehensive, practical, and structured exam preparation package that enables confident, accurate test performance.

## Source Materials to Use
1. **Provided PDFs or official course materials** - the authoritative core content
2. **Specific learning objectives and topics** - the explicit exam scope
3. **Topic emphasis** - areas marked, repeated, or expanded in materials indicate higher priority

---

## Student Learning Goals
The student needs to:

- **Understand deeply** - explain concepts intuitively AND formally; apply them in new contexts
- **Recognize accurately** - identify definitions, proof types, and concepts from descriptions, examples, problem contexts, and multiple-choice wording
- **Select strategically** - choose the right proof method for each problem type
- **Execute correctly** - write proofs with proper structure, valid logic, and clear communication
- **Distinguish precisely** - tell apart similar or easily-confused concepts
- **Perform confidently** - solve exam questions under time pressure

---

## Preparation Package Structure

### STEP 1: Material Review and Organization
- **Read thoroughly** - all PDFs and learning objectives
- **Identify all components** - major concepts, subtopics, definitions, proof types, theorems, methods, patterns, and recurring themes
- **Create a topic structure** - group logically and hierarchically
- **Flag discrepancies** - note topics emphasized in materials but not in objectives (and vice versa)
- **Assess depths** - determine which topics need surface-level vs. deep understanding

### STEP 2: Create Master Exam Overview
Produce a high-level overview that shows:
- **What exams tests** - main units/topic clusters and their relative importance
- **Depth expectations** - what must be memorized vs. understood vs. applied
- **Key content** - most important ideas students must know
- **Confusion zones** - where students typically struggle, what looks similar, what trips up test-takers
- **Emphasis signals** - which topics appear most frequently or are most heavily developed

### STEP 3: Build a Concise Cheat Sheet
Create a one-one-and-half-page reference containing:
- **Definitions** - exact, formal definitions of all key terms
- **Symbols and notation** - complete reference of all symbols used
- **Key theorems/facts** - important statements, formulas, logical rules
- **Proof types** - name, when to use, quick recognition clue
- **Logical equivalences** - important transformations, rules, relationships
- **"If you see X, think Y" reminders** - pattern recognition shortcuts
- **Common traps** - mistakes students make, incorrect reasonings to avoid
- **Quick formulas/facts** - anything that takes time to derive but might be needed fast

### STEP 4: Create Detailed Study Guide
For **every** learning objective and topic, provide:

**Plain-language explanation** - intuitive, conversational description of the concept that builds understanding from scratch

**Formal definition** - exact mathematical/logical statement of what this is

**Why it matters** - its role in the larger course structure and why students need to know it

**Problem contexts** - where and how this appears in exams and problem sets; what types of problems use it

**Multiple-choice recognition** - how to identify this concept in MCQs:
  - What phrasing signals this answer?
  - What are common wrong answers?
  - How is it distinguished from look-alike concepts?

**Detailed examples** - 2-3 worked examples showing the concept in action; vary problem types if applicable

**Common mistakes** - explicit list of incorrect approaches, misconceptions, wrong answers students give

**Memory aids** - memorable phrases, analogies, visual patterns that help recall

### STEP 5: Comprehensive Proof Strategy Guide

#### Part A: General Proof Framework
Provide a systematic method any student can apply:
- How to parse a proof problem
- How to translate mathematical language into actionable steps
- How to choose a proof type
- How to structure and write a proof
- How to verify correctness

#### Part B: Specific Methods by Proof Type
For **each relevant proof type**, provide:
- **What it is** - plain-language definition
- **When to use it** - problem indicators that suggest this method
- **Recognition clue** - how to spot when a problem wants this approach
- **Step-by-step method** - numbered procedure to follow
- **Sentence templates & structure** - opening, middle, closing formulas unique to this proof type
- **Common mistakes** - what students do wrong with this type
- **Small worked example** - one clear, complete example from start to finish

**Include these proof types** (or note if not relevant):
  - Direct proof
  - Proof by contrapositive
  - Proof by contradiction
  - Proof by cases / exhaustive cases
  - Existence proofs (finding a witness)
  - Uniqueness proofs (showing only one object satisfies)
  - Mathematical induction (single step)
  - Strong induction (multiple prior cases)
  - Biconditional proofs ("if and only if")
  - Set equality and subset proofs
  - Divisibility proofs
  - Disproof by counterexample

### STEP 6: Proof Selection Flowchart
Create a practical decision tree that helps choose the right proof type:
- Start with observable features of the statement
- Branch based on logical structure (implication, universal, existential, etc.)
- Offer decision criteria and "if stuck" alternatives
- Make it a reference tool a student could use during an exam

Example format:
```
Is this "for all integers n ≥ ..."? → YES → Try induction
Does next step depend on several prior cases? → YES → Use strong induction
Is this an implication P → Q? → YES → Try direct proof first...
```

### STEP 7: Definition Recognition & Multiple Choice Strategy

**Concept matching guide:**
- Organized alphabetically or by theme
- One-line definitions for rapid lookup
- Emphasis on how to distinguish from similar terms
- Symbols and notations

**Common look-alike pairs:**
- Contrapositive vs. Converse - exact distinction
- Example vs. Counterexample - when each applies
- Existence vs. Universality - what each claim requires
- Direct proof vs. Contradiction - different starting points
- Lemma vs. Corollary - different roles
- Strong vs. weak induction - when to use each
(For each pair: clear definition of each, one key difference, a short example if helpful)

**MCQ elimination strategies:**
- Spot-check answers against known traps
- Red flags that eliminate options immediately
- Pattern recognition that points to correct answer

### STEP 8: Active Recall Practice Materials

Create a substantial question bank including:

**Likely exam questions by topic** - realistic questions the exam may ask, organized by topic

**Concept check questions** - short-answer questions testing understanding:
  - "What is X?"
  - "Why does X matter?"
  - "How is X different from Y?"
  - "When would you use X?"

**Proof-style problems** - problems requiring proof construction, graded by difficulty

**Definition recognition questions** - matching, identification, and concept questions in multiple-choice style

**"Explain why this is wrong" questions** - given a student's incorrect proof or reasoning, identify the error

**Application questions** - problems that require combining multiple concepts

### STEP 9: Material Prioritization
Rank all topics in tiers:

**TIER 1 - Must Know Cold:**
Topics that are fundamental, heavily emphasized, or prerequisite to other topics. These appear frequently in exams. Study these first; master completely.

**TIER 2 - Very High Priority:**
Topics that are important and likely to appear but not foundational. Study these early in preparation.

**TIER 3 - Important but Specialized:**
Topics with narrower application or lower frequency. Study after Tier 1-2 are solid.

For each tier, briefly explain why it has that ranking.

**Time allocation strategy:**
- If you have 1-2 weeks: Tier 1 only + practice tests
- If you have 3-4 weeks: Tier 1 + Tier 2 + practice
- If you have 5+ weeks: All tiers + deep practice + corrections

### STEP 10: Final Exam Resources

**One-page night-before exam summary:**
- Absolute essentials - definitions and facts you need to recall
- Critical proof structures and templates
- Top 3 most common student errors and how to avoid them
- Quick checklist for proof verification

**How to approach proof questions under time pressure:**
- Quick parsing technique
- Rapid proof type selection
- Fallback strategies if you get stuck
- How to write a partial proof for partial credit

**How to approach multiple-choice definition questions:**
- Elimination strategies
- Red flag phrases to avoid
- When to guess vs. skip
- Pattern recognition shortcuts

## Execution Standards

**Clarity:** Use clean headings, organized structure, and consistent formatting. Text should be readable and scannable.

**Accuracy:** Base everything directly on provided materials. No vague advice. Every claim should be traceable to the source material.

**Completeness:** Do not compress or skip sections. If material is large, expand explanations rather than condense. Err on the side of too much detail rather than too little.

**Dual explanations:** Explain both intuitively (narrative, analogy, context) AND formally (definitions, notation, logic).

**Relevance flagging:** If a proof type isn't applicable, state that clearly. If a topic is heavily emphasized or appears frequently, flag it.

**Common pitfalls:** Include a specific section on mistakes; don't assume students see them naturally.

**Gap highlighting:** Point out where students are likely to have incomplete understanding, even if they can solve routine problems.

4. **Proof Strategy Guide** - Method + specific techniques
5. **Proof Selection Flowchart** - Decision tree
6. **Definition Recognition Guide** - Term identification
7. **Practice Questions** - Active recall materials
8. **Prioritization Plan** - Study strategy
9. **Night Before Summary** - Ultra-condensed essentials
10. **Time Pressure Strategies** - Exam execution tips

---

## Special Instructions

- **Do not skip proof strategy.** This is the heart of exam success.
- **Be concrete, not vague.** Every explanation should be followed by an example.
- **Point out confusion zones explicitly.** Don't assume students see what trips them up.
- **Repeat the most important ideas.** Key concepts should appear in cheat sheet, study guide, practice questions, and summary.
- **Use sentence templates liberally.** Students often freeze on how to start and end proofs. Provide exact language they can adapt.
- **Clearly flag relevance gaps.** If something appears in material but not in objectives (or vice versa), call it out and explain how you handled it.
- **Err on the side of completeness.** Large material should be explained thoroughly, not compressed.
