## **STEP 5: COMPREHENSIVE PROOF STRATEGY GUIDE**

This is the heart of exam success. Include two major parts:

### **PART A: Universal Proof Framework**

Create a step-by-step process ANY student can follow for ANY proof problem:

**STEP 1: Parse the Problem**
- Identify the statement form: simple implication? Universal? Existential? Biconditional? Equivalence chain?
- Point to key logical words: for all, there exists, if...then, if and only if, not, and, or
- Translate into plain English

**STEP 2: Identify What You Need to Prove**
- Highlight the premise (what you may assume)
- Highlight the conclusion (what you must show)
- Write this explicitly: "We must show: [exact statement]"

**STEP 3: Write Down All Relevant Definitions**
- Define every term used in the statement
- Algebraically (even=2k, divisible=mk, etc.)
- This is NON-NEGOTIABLE
- Leave space to use these definitions during proof

**STEP 4: Choose Your Proof Source (Flowchart Reference)**
Use the flowchart below (STEP 6) to select best method

**STEP 5: Write Your Opening Line**
Don't start with algebra. Start with structure:
- "We proceed by direct proof."
- "We prove by contrapositive."
- "We proceed by cases."
- Etc.

**STEP 6: Prove, Step by Step**
- Each line should move toward conclusion
- Justify each step (use definition, arithmetic, prior step)
- Refer explicitly to IH in induction proofs
- Don't skip steps (even "obviously true" steps)

**STEP 7: State Your Conclusion**
- Restate what you proved
- Make clear you've completed the required task
- Usually: "Therefore [exact statement]."

**STEP 8: Verify (Checklist)**
For proofs, after writing:
- Did I start from the right assumption?
- Did I prove exactly what was asked?
- Is every step justified?
- Is the logic flow clear?
- Would someone unfamiliar understand this?

## **STEP 6: PROOF SELECTION FLOWCHART**

Create a practical, exam-ready decision tree that a student could actually use under time pressure.

Format as nested questions:

```
START: Read the statement carefully.
|
├─ Is this "for all n ≥ n₀" or "for every integer n"?
│  ├─ YES → Try INDUCTION
│  │        ├─ Does the next case depend on multiple prior cases?
│  │        │  ├─ YES → Use STRONG INDUCTION
│  │        │  └─ NO → Use regular induction
│  │
│  └─ NO → Continue to next question
│
├─ Is this an implication (P → Q) or conditional?
│  ├─ YES →
│  │        1. FIRST TRY: Direct proof (assume P, prove Q)
│  │        2. IF STUCK: Contrapositive (assume ¬Q, prove ¬P)
│  │        3. IF VERY STUCK: Contradiction (assume P ∧ ¬Q, derive false)
│  │
│  └─ NO → Continue
│
├─ Is this "if and only if" or "iff"?
│  ├─ YES → BICONDITIONAL (prove both directions)
│  │
│  └─ NO → Continue
│
├─ Is this about sets (membership, subset, equality)?
│  ├─ YES →
│  │        ├─ x ∈ A? Prove defining property
│  │        ├─ A ⊆ B? Let x ∈ A, show x ∈ B
│  │        └─ A = B? Prove both A ⊆ B and B ⊆ A
│  │
│  └─ NO → Continue
│
├─ Does it say "there exists"?
│  ├─ YES → EXISTENCE PROOF (find a witness/example)
│  │        ├─ Must also show uniqueness?
│  │        │  └─ YES → Follow with uniqueness argument
│  │
│  └─ NO → Continue
│
├─ Does it say "no such object exists" or claim falsity?
│  ├─ YES →
│  │        ├─ Use CONTRADICTION (assume it exists, derive false)
│  │        ├─ OR use COUNTEREXAMPLE if disproving a universal claim
│  │
│  └─ NO → Continue
│
└─ Are there natural split cases (even/odd, positive/negative/zero)?
   ├─ YES → PROOF BY CASES (cover each case exhaustively)
   │
   └─ NO → You may have missing context; re-read problem
```

**IF STUCK:**
- Rewrite statement using definitions (don't be abstract)
- Test a few small examples and look for patterns
- Negate the conclusion and see if contrapositive is easier
- Ask: Is this really "for all"? "There exists"? "if-then"?
- Try all three (direct, contrapositive, contradiction)

### **10b. Proof Question Exam Strategy**
Teach the student how to write proofs under time pressure:

**Time T = 0-30 seconds: Parse**
- Read statement twice
- Highlight hypothesis and conclusion
- Identify statement type (implication? existence? etc.)

**Time T = 30-60 seconds: Plan**
- Write down definitions
- Choose proof type using flowchart
- Jot outline of proof (2-3 lines of structure)
- Identify first and last sentences

**Time T = 60-180 seconds: Write**
- Write structured proof
- Don't over-explain (you have limited time)
- Every step justified but concise
- If unsure on algebra, keep going (partial credit)

**If Stuck:**
- Write proof outline/idea even if incomplete
- Show definitions and reasoning
- Avoid blanks (even wrong work shows thinking)

**If Running Out of Time:**
- Finish proof-in-outline form
- Write answers to easier parts fully
- Bullet key remaining steps

### **10c. Multiple-Choice Exam Strategy**
How to approach MCQ questions efficiently and accurately:

**For Each MCQ:**
1. **Read fully** (all options before deciding)
2. **Eliminate** (mark obviously wrong)
3. **Analyze remaining** (test against definitions)
4. **Use elimination checklist:**
   - Does it match our definitions?
   - Does it avoid known traps?
   - Can I justify why it's correct?
5. **Decide or skip** (come back if time)

**Red Flag Phrases (Auto-Eliminate):**
- "One example proves for all"
- "Converse = contrapositive"
- "Only need one direction of if-and-only-if"
